# 代码生成指令

请严格参考以下已确认文档：

1. [PRD](./01_PRD.md)
2. [架构文档](./02_ARCH.md)
3. [接口文档](./03_API_DOC.md)
4. [实现规范](./04_SPEC.md)

在此基础上实现【XX 模块】。

## 强制约束

- 结构必须与文档一致，不允许新增未定义逻辑。
- 只能实现文档中已经定义的模块、字段、接口、状态码和数据流。
- 若新增接口，必须先补 `PRD -> ARCH -> API Doc -> SPEC -> TEST CASE`，再写代码。
- 注释保持简洁，只解释复杂分支、数学逻辑或兼容性处理。
- 如果实现涉及自定义 Provider，必须先核对对应官方文档；发现线上返回异常时，不允许跳过文档核验直接硬改解析逻辑。
- 新增 Provider 时，必须同步更新：
  - `providers/__init__.py` 注册表
  - 测试文件
  - `docs/project_docs/` 下相关文档

## 交付要求

- 明确列出本次实现对应的文档章节。
- 代码返回结构必须与 [API Doc](./03_API_DOC.md) 一致。
- 目录位置必须与 [ARCH](./02_ARCH.md) 和 [SPEC](./04_SPEC.md) 一致。
- 不允许在未更新 [TEST CASE](./05_TEST_CASE.md) 的情况下新增测试范围。
- 若改动 Provider 取数逻辑，交付说明必须包含“官方文档来源 + 实时验证结论 + 结果抽检结论”。
