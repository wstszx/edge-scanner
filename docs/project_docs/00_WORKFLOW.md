# Edge Scanner 文档工作流

项目名称：Edge Scanner

## 1. 项目启动指令

后续如需继续让 AI 扩展本项目，请先发送以下指令：

```text
请严格按照以下流程执行：

1. 先写 PRD
2. 再写架构文档
3. 再写接口文档
4. 再写实现规范
5. 再写测试用例
6. 再写代码
7. 再写测试
8. 最后写项目总结
全程必须参考前面已确认的文档，不允许自由发挥。
```

## 2. 执行约束

- 文档顺序固定：`PRD -> ARCH -> API -> SPEC -> TEST CASE -> CODE -> TEST -> SUMMARY`
- 后置文档必须引用前置文档，不允许新增未定义字段、接口、流程或模块。
- 当前仓库代码已存在，本轮产出以“反向归档现有实现”为准，不对业务逻辑作额外假设。
- 代码和测试扩展时，必须优先更新本目录下文档，再实施变更。

## 3. 本地文档清单

- `01_PRD.md`
- `02_ARCH.md`
- `03_API_DOC.md`
- `04_SPEC.md`
- `05_TEST_CASE.md`
- `06_CODE_GENERATION_PROMPT.md`
- `07_TEST_GENERATION_PROMPT.md`
- `08_PROJECT_SUMMARY.md`

## 4. 一句话工作流

```text
PRD -> 架构 -> 接口 -> 规范 -> 测试用例 -> 代码 -> 测试 -> 总结
全程强制参考文档，AI 无自由发挥权。
```
